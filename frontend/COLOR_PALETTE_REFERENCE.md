## Accessibility Compliance

### Contrast Ratios
- **Text Primary on White**: 16.8:1 (AAA)
- **Text Secondary on White**: 4.5:1 (AA)
- **Primary Green on White**: 4.5:1 (AA)
- **Accent Green on White**: 3.5:1 (AA)

### WCAG Compliance
- All text colors meet minimum WCAG AA standards
- Most combinations exceed AA, many reach AAA
- Dark mode maintains proper contrast ratios

### High Contrast Mode
- Automatic detection via `prefers-contrast: more`
- Increased border visibility
- Underlined links for clarity
- Maintained readability across all elements

## Gradient Usage

### Brand Gradient
```css
background: linear-gradient(135deg, #16A34A 0%, #22C55E 100%);
```
Usage: Headers, hero sections, accent elements

### Gradient Text
```css
background: linear-gradient(135deg, #16A34A 0%, #22C55E 100%);
-webkit-background-clip: text;
-webkit-text-fill-color: transparent;
background-clip: text;
```
Usage: Emphasis text, titles, brand statements

### Dark Mode Gradient
```css
background: linear-gradient(135deg, #86EFAC 0%, #22C55E 100%);
```
Usage: Same as brand gradient, adapted for dark mode

## Implementation Examples

### Primary Button
```tsx
<Button variant="contained" sx={{ backgroundColor: '#16A34A' }}>
  Primary Action
</Button>
```

### Secondary Button
```tsx
<Button variant="outlined" sx={{ borderColor: '#E5E7EB', color: '#16A34A' }}>
  Secondary Action
</Button>
```

### Success Alert
```tsx
<Alert severity="success" sx={{ backgroundColor: '#ECFDF5', borderColor: '#86EFAC' }}>
  Operation successful!
</Alert>
```

### Card
```tsx
<Card sx={{
  borderRadius: '12px',
  border: '1px solid #E5E7EB',
```

## Customization

### To Change Brand Color
1. Update `COLORS.primary` in `main.tsx`
2. Update corresponding dark/light variants
3. Update CSS selectors in `index.css`
4. Test contrast ratios for accessibility

### To Add New Color
1. Add to `COLORS` object in `main.tsx`
2. Add Material-UI palette entry if applicable
3. Document in this reference
4. Test accessibility and dark mode support
